## ğŸ› Troubleshooting

### "Workflow not running"

**Check:**
1. Workflow file exists: `.github/workflows/deploy-docs.yml`
2. GitHub Actions enabled: Settings â†’ Actions â†’ Allow all actions
3. File permissions: `git ls-files --stage .github/workflows/deploy-docs.yml`

**Fix:**
```bash
git add .github/workflows/deploy-docs.yml
git commit -m "Fix workflow"
git push
```

### "Build failing"

**Check the logs:**
1. Actions tab â†’ Click failed run
2. Read error message

**Common issues:**

**Error: "Doxyfile not found"**
```bash
# Make sure Doxyfile is committed
git add Doxyfile
git commit -m "Add Doxyfile"
git push
```

**Error: "Permission denied"**
```bash
# Check repository Settings â†’ Actions â†’ Workflow permissions
# Should be: "Read and write permissions"
```

**Error: "Page not deploying"**
```bash
# Check Settings â†’ Pages
# Source should be: "GitHub Actions"
```

### "Documentation outdated"

**Force rebuild:**

1. Go to Actions tab
2. Click "Deploy Documentation" workflow
3. Click "Run workflow" button (top right)
4. Click "Run workflow" (green button)

Manual trigger forces fresh build.

### "404 on GitHub Pages"

**Cause:** GitHub Pages not enabled or wrong source.

**Fix:**
1. Settings â†’ Pages
2. Source: **GitHub Actions** (not "Deploy from a branch")
3. Save
4. Wait 5 minutes, refresh

## ğŸ“ Workflow File Explained

The `.github/workflows/deploy-docs.yml` file does this:

```yaml
# When to run
on:
  push:
    branches: [ main ]        # Run on push to main
    paths:                    # Only if these files change
      - 'main.c'
      - 'docs/**'
      - 'Doxyfile'

# What to do
jobs:
  build-docs:
    steps:
      - Checkout code
      - Install Doxygen
      - Run: doxygen Doxyfile
      - Upload docs/html/ to GitHub Pages
      - Deploy!
```
